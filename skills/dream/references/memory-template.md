# MEMORY.md Template

```markdown
# Project Memory

## Rules
- [Project-specific rules discovered during work]

## Architecture Decisions
- [Key decisions with rationale]

### {Decision Name}
- **Date**: YYYY-MM-DD
- **Context**: Why this was decided
- **Decision**: What was chosen
- **Alternatives considered**: What was rejected and why

## Patterns
- [Established conventions]

### {Pattern Name}
- **Description**: What the pattern is
- **Example**: How to apply it
- **When to use**: Applicable contexts

## Learnings
- [Bug fixes, gotchas, insights]

### {Learning Name}
- **Problem**: What went wrong
- **Root cause**: Why it happened
- **Solution**: How it was fixed
- **Prevention**: How to avoid it

## Preferences
- [User style, constraints]

### {Preference Name}
- **Description**: What the preference is
- **Context**: When it applies
```

## Size Guidelines

- Keep MEMORY.md under 500 lines
- Each section: max 50 lines
- Each entry: max 10 lines
- Use links to detailed docs if needed

## Maintenance

- Review quarterly
- Archive outdated entries to `archive/memory/`
- Keep only actionable, stable knowledge
```
